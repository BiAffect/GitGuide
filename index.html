Git Tutorial

Step 1: Clone the repository

Use the following command to clone the repository to your local machine:

git clone https://github.com/BiAffect/SynapseDataProcessing.git

                
cd SynapseDataProcessing

Step 2: Pull existing scripts

Before making changes, ensure you have the latest version of the repository on your local machine:

git pull origin main

Step 3: Make changes and commit

Make your changes to the file that may cause a merge conflict.

Add the changes to the staging area and commit them with a descriptive message:

git add .

                
git commit -m "Your commit message here"

Step 4: Push changes to GitHub

Push your committed changes back to the shared GitHub repository:

git push origin main

Step 5: Resolve merge conflicts

If there are multiple users making changes to the same file, it can lead to merge conflicts.

When you encounter merge conflicts, Git will indicate which files have conflicts. Open the conflicting file(s) in a text editor and resolve the conflicts manually.

The conflicting lines will be marked, and you'll need to choose which version of the code to keep.

After resolving the conflicts, add the resolved changes to the staging area and commit them:

git add .

                
git commit -m "Merge conflict resolved"
